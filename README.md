# LLM内容合规审计插件
一键阻止大模型被诱导输出违规消息，AstrBot 专属的内容合规检测插件，为机器人提供双重安全防护。

## 功能特性
✅ 输出合规检测：默认开启，LLM生成回复后自动检测，不合规则替换为拦截提示
✅ 输入合规检测：可选开启，用户发送消息后直接检测，不合规则不调用LLM
✅ 正则黑白名单：支持自定义违禁词/白名单正则规则，灵活控制拦截范围
✅ 自定义合规表达：支持添加允许的词汇，自动加入白名单并告知大模型放行
✅ 管理员动态添加：支持通过指令快速添加合规表达，无需修改配置重启插件
✅ 自定义审核模型：可单独指定输入/输出检测使用的大模型，适配不同场景
✅ 会话忽略：支持指定会话跳过所有检测，适配信任的群聊/私聊场景
✅ 自定义拦截回复：可自由设置内容被拦截时发送给用户的提示语
✅ 自定义审核规则：支持修改大模型检测提示词，适配个性化合规要求

## 安装方法
1. **在线安装**：在 AstrBot WebUI 插件管理页面，粘贴本仓库地址，点击安装即可
2. **手动安装**：将本插件所有文件放入 `data/plugins/astrbot_plugin_llm_Output_audit` 目录，重载插件即可

## 配置说明
所有配置均可在 AstrBot WebUI 的插件配置页面可视化修改，无需手动编辑文件。

### 核心配置项
1. **启用用户输入检测**：是否在用户发消息时就进行合规检测
2. **输入/输出检测模型**：选择用于合规审核的大模型，不选则使用当前会话默认模型
3. **违禁词正则列表**：填写正则规则，匹配到的内容直接拦截
   - 示例：`色情|低俗|裸聊`、`刷单|返利|中奖`、`1[3-9]\d{9}`
4. **白名单正则列表**：匹配到的内容跳过违禁词检测，优先级高于违禁词
   - 示例：`XX官方旗舰店`、`12345678901`
5. **合规的表达列表**：格式为「词汇~原因」，用~分隔词汇和合规原因
   - 示例：`我去~表达惊讶的日常语气词`、`卧槽~表达强烈情绪的口语`
   - 这些内容会自动加入白名单，同时告知大模型为合规内容，不会被拦截
6. **忽略会话列表**：填写会话的 `unified_msg_origin`（发送 `/sid` 命令可获取）
7. **拦截回复内容**：内容被拦截时发送给用户的提示语
8. **检测提示词**：自定义大模型的审核规则，非专业用户不建议修改

## 管理员指令
仅机器人管理员可使用，添加后实时生效，无需重启插件。

### 添加合规表达
- 指令格式：`加合规 词汇 合规原因`
- 别名：`添加合规`、`新增合规`
- 示例：
  - `加合规 我去 表达惊讶的日常语气词`
  - `加合规 卧槽 表达强烈情绪的口语化表达`
- 注意：词汇和原因中不能包含分隔符「~」，原因支持带空格的长文本

## 注意事项
1. 正则表达式请确保语法正确，错误的规则会被自动跳过并在日志中提示
2. 大模型检测效果依赖所选模型的能力，建议使用能力较强的模型进行审核
3. 合规表达的白名单仅对完整词汇生效，不会放行包含该词汇的违规内容
4. 插件所有日志都会以 `llm_output_audit` 为标识输出，方便排查问题
